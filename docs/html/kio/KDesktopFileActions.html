KDesktopFileActions provides a number of methods related to actions in desktop files.

  • run (_url, _is_local)
    returns (bool)
    ArgumentTypeDefault
    _urlKUrl
    _is_localbool

    Invokes the default action for the desktop entry. If the desktop entry is not local, then only false is returned. Otherwise we would create a security problem. Only types Link and Mimetype could be followed.

    _url - the url to run

    _is_local - true if the URL is local, false otherwise Returns true on success and false on failure.

    See also KRun.runUrl

  • userDefinedServices (path, bLocalFiles)
    returns (list of KDesktopFileActions.Service)
    ArgumentTypeDefault
    pathQString
    bLocalFilesbool

    Returns a list of services defined by the user as possible actions on the given .desktop file. May include empty actions which represent where visual separators should appear in user-visible representations of those actions, such as separators in a menu.

    path - the path to the desktop file describing the services

    bLocalFiles - true if those services are to be applied to local files only (if false, services that don't have %u or %U in the Exec line won't be taken into account). Returns the list of user deviced actions
