Class KConfig
Inherits KConfigBase

Access KDE Configuration entries.

This class implements KDE's default configuration system.

Author Kalle Dalheimer , Preston Brown

See also KGlobal.config(), KConfigBase, KConfigGroup KDE Configuration Management class

  • checkUpdate (id, updateFile)
    returns nothing
    ArgumentTypeDefault
    idQString
    updateFileQString

    Checks whether the config file contains the update id as contained in updateFile. If not, it runs kconf_update to update the config file.

    If you install config update files with critical fixes you may wish to use this method to verify that a critical update has indeed been performed to catch the case where a user restores an old config file from backup that has not been updated yet.

    id - the update to check

    updateFile - the file containing the update

  • lockFile (bGlobal)
    returns (KLockFile.Ptr)
    ArgumentTypeDefault
    bGlobalboolFalse

    Returns a lock file object for the configuration file or 0 if the backend does not support locking.

    bGlobal - if true, return the lock file for the global config file

    NOTE: KConfig.sync() requires a lock on both the normal and global config file. When calling KConfig.sync() while having a lock on the global config file, the normal config file MUST be locked AS WELL and the normal config file MUST be locked BEFORE the global config file! Otherwise there is a risk of deadlock.
